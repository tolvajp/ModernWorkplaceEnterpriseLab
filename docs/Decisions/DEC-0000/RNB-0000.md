# Runbook – Platform registration (Microsoft Intune)

## Status
Executed

## Objective
- This runbook describes how to create a new Microsoft cloud tenant to establish an Intune-capable environment.  
- No endpoint configuration or device enrollment is performed as part of this runbook.

---

## Scope

### In Scope

- Microsoft cloud tenant registration
- Initial administrative access verification
- Platform availability validation

### Out of Scope

- Any configuration on the tenant beyond initial registration

---

Conditions that **must be true** before execution.

## Risks & Safety Notes

## Procedure

### Step 1 – Preparation

- [x] External email address available for tenant registration
- [x] Debit/credit card for tenant registration
- [x] Tenant parameters defined in `RNB-0000.json`


---

### Step 2 – Execution
Describe the core operational steps in **conceptual terms**.

- Register an external email address using the value defined in `RNB-0000.json` (`registrationEmail`)
- Register a new Microsoft cloud tenant using the parameters defined in `RNB-0000.json`
- Complete mandatory verification and consent steps
- Do not configure any tenant settings beyond initial registration


---

### Step 3 – Validation

- Log in to the newly created tenant
- Verify access to Microsoft Entra admin center
- Verify access to Microsoft Intune admin center
