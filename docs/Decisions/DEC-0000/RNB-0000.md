# Runbook – Platform registration (Entra ID + Intune foundation)

## Status
Executed

---

## Preconditions
Conditions that **must be true** before execution.

- [x] Related decision (DEC-0000) is in status **Accepted**
- [x] External email address available for tenant registration
- [x] Debit/credit card available if required for trial/verification
- [x] Tenant parameters defined in `RNB-0000.json`
- [x] No conflicting runbook already executed for the same tenant name

---

## Risks & Safety Notes
- Tenant creation is not reversible without deletion and a mandatory waiting/retention period
- Incorrect tenant naming may require recreation
- Trial/subscription limitations may apply depending on region and eligibility
- Do not perform any configuration outside the explicit scope of this runbook

---

## Procedure

### Step 1 – Preparation
- Review DEC-0000 scope and confirm this runbook only performs **registration + access validation**
- Confirm `registrationEmail` and `tenantName` values in `RNB-0000.json`
- Ensure you can receive verification codes for the registration email

---

### Step 2 – Execution
- Register a new Microsoft cloud tenant using the parameters in `RNB-0000.json`
- Complete mandatory verification and consent steps
- Create or confirm a global admin identity for initial access (as required by the registration flow)
- Do **not** configure tenant settings beyond initial registration

---

### Step 3 – Validation
- Log in to the newly created tenant
- Verify access to:
  - Microsoft Entra admin center
  - Microsoft Intune admin center
- Record the tenant identifiers and any relevant evidence (screenshots/notes) if required
